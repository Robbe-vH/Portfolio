Competenties

Ontwerper

Waar sta je momenteel?

Als ontwerper heb ik nog weinig ervaring, deze website is een van de grotere projecten die ik zelf heb kunnen ontwerpen. Het zelf kiezen van een ontwerp vind ik wel leuk, maar creativiteit is niet mijn sterkste punt.

Wat lukt je al goed?

Het kiezen van kleuren en opdelen van pagina's met samenhangende opmaak lukt vrij goed.

Wat lukt je nog niet?

Originele ontwerpen maken die uit de massa springen is niet mijn sterkste kant, het ontwerp van mijn website is al door duizenden mensen voor mij gedaan.

Voorbeelden

  • Tijdens het Webproject van WPL 1 hebben we ons eigen ontwerp kunnen kiezen en uitwerken.
  • Tijdens het C# project is er ook gevraagd om een eigen ontwerp te maken.
  • Later zullen we denk ik vaker moeten luisteren naar een UX/UI designer die ons de foto's en de rest van het ontwerp uittekent.

Programmeur

Waar sta je momenteel?

Als programmeur heb ik mijn best gedaan de eerste maanden om veel kennis binnen en buiten school op te doen. Na mijn uren op school heb ik me verdiept in andere talen en projecten. Dit heeft me veel geholpen om de leerstof in de vingers te krijgen. Ook heb ik veel moeite gestoken in het opdrachten gedeelte van de website. Hier is een C# spel gemaakt in Unity in de website verwerkt.

Wat lukt je al goed?

Leerstof van de lessen toepassen in de projecten van WPL, eigenhandig nieuwe dingen opzoeken en toepassen.

Wat lukt je nog niet?

Snel en efficient door de documentatie van talen zoeken en de voorbeelden gemakkelijk omzetten naar mijn eigen programma.

Voorbeelden

  • Bij de C# opdracht werden er uitbreidingen gevraagd die we nog niet in de les hadden gezien, dus daardoor zijn we eigenhandig moeten gaan opzoeken.
  • Bij onze portfolio opdracht zjin we vrij geweest om skills van buiten de les toe te passen, daarom heb ik de beginselen van Angular geleerd en deze website ermee gemaakt.
  • Later zullen er dagelijks dingen komen die we nog niet hebben gezien, daarom is het nu belangrijk om effecient zelf nieuwe documentatie te kunnen lezen en begrijpen.

Tester

Waar sta je momenteel?

Om code te testen heb ik SonarQube opgesteld in een docker-container op mijn server thuis. Dit was leuk om mee te experimenteren en om feedback te krijgen op mijn code. Zelf heb ik ook vaak mijn C# BlackJack spel moeten testen om te kijken of alles werkt zoals het hoort.

Wat lukt je al goed?

Basis code overlopen om fouten op te sporen.

Wat lukt je nog niet?

Effecient op zoek gaan naar waar de fout zich precies voor doet. Vooral bij C# vind ik dit een moeilijkere opgave dan bij Javascript/Typescript.

Voorbeelden

  • Het debuggen van het BlackJack spel.
  • Het werkend krijgen van het Interactief Portfolio spel in een Angular website had redelijk veel debugging nodig.
  • Later zullen we ook constant onze eigen code moeten nalezen en debuggen, tenzij dat er een gespecialiseerde tester aanwezig is, of als er een server opstaat met scripts om dit te automatiseren.

Communicator/Teamspeler

Waar sta je momenteel?

We hebben tot nu toe nog geen grote groepswerken gehad. Maar van mezelf weet ik dat ik wel graag in groep werk, als de rest ook goed mee werkt aan het project.

Wat lukt je al goed?

Werken met Git begint goed te klikken. Dit lijkt me een van de belangrijkere tools in het werken als een team.

Wat lukt je nog niet?

Aangezien we nog niet echt een groepswerk hebben moeten doen weet ik niet zeker wat me niet goed lukt, maar ik kan me inbeelden dat ik niet goed ben in mensen te motiveren die niks doen in een groepswerk.

Voorbeelden

  • Later zullen we vaak in groep moeten werken, agile en scrum werken zal een belangrijk onderdeel zijn.
  • Het leren inschatten van user stories op basis van hun intesiteit zal ook wat oefening vergen.
  • Later zal het ook een uitdaging zijn om te werken met een team wat voor de helft misschien van thuis uit werkt.

Levenslang lerende IT-Professional

Waar sta je momenteel?

Ik doe buiten mijn lessen regelmatig een sessie op PluralSight om nieuwe skills te leren. Het interesseert me wel om andere frameworks te leren kennen en om te kijken welke software paketten er bestaan. Ook lijkt het me nuttig om dingen bij te leren over systemen en netwerken.

Wat lukt je al goed?

Nieuwe dingen bijleren via het internet, vragen stellen op bv. StackOverflow en Reddit.

Wat lukt je nog niet?

Efficient door documentatie bladeren en voorbeelden toepassen.

Voorbeelden

  • Als IT-er zal je altijd moeten blijven bijleren om mee te zijn met de nieuwe technologie.
  • Later zal er ok van je verwacht worden om een framework bij te leren op een paar dagen om een opdracht te kunnen maken, daarom is het nuttig om alvast eentje te snappen.
  • Het lijkt me normaal om interesse te hebben in andere vakgebieden van IT zodat je de volledige werking snapt en niet alleen het programmeer gedeelte
