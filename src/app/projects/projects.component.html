WPL2

Webshop "For Rent"

Dit project omvatte de ontwikkeling van een full-stack website voor het huren en verhuren van luxe voertuigen. Het systeem was opgebouwd uit verschillende componenten, waaronder een MicroSoft SQL Server database, een .NeT web API en een front-end in Vue.js. De website bood gebruikers de mogelijkheid om luxe voertuigen te zoeken, huren en verhuren. Het omvatte functies zoals het bekijken van beschikbare voertuigen, het plaatsen van reserveringen en het beheren van gebruikersprofielen. De front-end, ontwikkeld in Vue.js, bood een intuïtieve gebruikersinterface waar gebruikers voertuigen konden bekijken, filteren op basis van criteria, beschikbaarheid controleren en reserveringen plaatsen. Het bood ook een gebruiksvriendelijk dashboard voor gebruikers om hun profielen te beheren en reserveringsinformatie te volgen. De database werd gebruikt om informatie over voertuigen, gebruikers en reserveringen op te slaan. De web API zorgde voor de communicatie tussen de front-end en de database, waardoor gegevens opgehaald en bijgewerkt konden worden. Ook zijn we tijdens dit project aan de slag gegaan als een scrum team en hebben we veel bijgeleerd over het samenwerken in een groep.

Competenties

Ontwerper

Waar sta je momenteel?

Als ontwerper heb ik nog weinig ervaring, deze website is een van de grotere projecten die ik zelf heb kunnen ontwerpen. Het zelf kiezen van een ontwerp vind ik wel leuk, maar creativiteit is niet mijn sterkste punt. Ik heb in dit project wel veel geleerd over kleurcombinaties, vormgeving van website elementen en een overzichtelijk design bewaren. Voor de kleuren te kiezen heb ik gebruik gemaakt van de Adobe Colour Picker website. In de toekomst hoop ik me te kunnen verdiepen in originelere ontwerpen die er minder basic uitzien.

Wat lukt je al goed?

Het kiezen van kleuren en opdelen van pagina's met samenhangende opmaak lukt vrij goed.

Wat lukt je nog niet?

Originele ontwerpen maken die uit de massa springen is niet mijn sterkste kant, het ontwerp van mijn website is al door duizenden mensen voor mij gedaan.

Voorbeelden

  • Tijdens het Webproject van WPL 1 hebben we ons eigen ontwerp kunnen kiezen en uitwerken.
  • Tijdens het C# project is er ook gevraagd om een eigen ontwerp te maken.
  • Later zullen we denk ik vaker moeten luisteren naar een UX/UI designer die ons de foto's en de rest van het ontwerp uittekent.

Programmeur

Waar sta je momenteel?

Als programmeur heb ik mijn best gedaan de eerste maanden om veel kennis binnen en buiten school op te doen. Na mijn uren op school heb ik me verdiept in andere talen en projecten. Dit heeft me veel geholpen om de leerstof in de vingers te krijgen. Ook heb ik veel moeite gestoken in het opdrachten gedeelte van de website. Hier is een C# spel gemaakt in Unity in de website verwerkt. Ook heb ik veel bijgeleerd over het Angular Framework doorheen dit project door gebruik te maken van Pluralsight.

Wat lukt je al goed?

Leerstof van de lessen toepassen in de projecten van WPL, eigenhandig nieuwe dingen opzoeken en toepassen.

Wat lukt je nog niet?

Snel en efficiënt door de documentatie van talen zoeken en de voorbeelden gemakkelijk omzetten naar mijn eigen programma.

Voorbeelden

  • Bij onze portfolio opdracht zjin we vrij geweest om skills van buiten de les toe te passen, daarom heb ik de beginselen van Angular geleerd en deze website ermee gemaakt.
  • Later zullen er dagelijks dingen komen die we nog niet hebben gezien, daarom is het nu belangrijk om effecient zelf nieuwe documentatie te kunnen lezen en begrijpen.

Tester

Waar sta je momenteel?

Om code te testen heb ik een SonarQube instantie opgesteld in een Docker-container op mijn server thuis. Dit was leuk om mee te experimenteren met Docker en om feedback te krijgen op mijn code. Zelf heb ik ook vaak mijn C# BlackJack spel moeten testen om te kijken of alles werkt zoals het hoort. Hieruit heb ik geleerd dat er altijd verbeteringen zijn aan te brengen in de code en dat vaak een probleem oplossen lijdt tot een ander probleem.

Wat lukt je al goed?

Basis code overlopen om fouten op te sporen.

Wat lukt je nog niet?

Effecient op zoek gaan naar waar de fout zich precies voor doet. Vooral bij C# vind ik dit een moeilijkere opgave dan bij Javascript/Typescript.

Voorbeelden

  • Het debuggen van het BlackJack spel.
  • Het werkend krijgen van het Interactief Portfolio spel in een Angular website had redelijk veel debugging nodig.
  • Later zullen we ook constant onze eigen code moeten nalezen en debuggen, tenzij dat er een gespecialiseerde tester aanwezig is, of als er een server opstaat met scripts om dit te automatiseren.

Wat lukt je al goed?

Wat lukt je nog niet?

Aangezien we nog niet echt een groepswerk hebben moeten doen weet ik niet zeker wat me niet goed lukt, maar ik kan me inbeelden dat ik niet goed ben in mensen te motiveren die niks doen in een groepswerk.

Voorbeelden

  • Later zullen we vaak in groep moeten werken, agile en scrum werken zal een belangrijk onderdeel zijn.
  • Het leren inschatten van user stories op basis van hun intesiteit zal ook wat oefening vergen.
  • Later zal het ook een uitdaging zijn om te werken met een team wat voor de helft misschien van thuis uit werkt.

WPL2

Het gebruik van WebAPI's in de webshop vergrootte mijn begrip van hun mogelijkheden en hielp bij implementeren van complexere functionaliteiten, zoals betalingsverwerking en communicatie met een database. Ik leerde ook het belang van timemanagement door doelen te stellen, plannen van mijn werk en prioriteren van taken, wat resulteerde in een vrij geslaagde afronding van het project binnen de gestelde tijdslimiet. Deze ervaring heeft mijn vermogen verbeterd om agile/scrum-methodologieën toe te passen, effectief samen te werken. Tijdens het project over de webshop heb ik waardevolle lessen geleerd over agile/scrum en teamwerk, waaronder effectief projectbeheer en aanpassing aan veranderende omstandigheden. Werken in sprints en het regelmatig houden van feedbacksessies bevorderde snelle vooruitgang en aanpassingen. Samenwerken met mijn teamleden versterkte mijn communicatie- en samenwerkingsvaardigheden, waardoor we efficiënter konden werken en beter konden inspelen op uitdagingen. Het is over het algemeen niet slecht verlopen, iedereen toonde inzet en was bereid om thuis wat te werken aan het project. Er zijn dingen niet afgeraakt die we graag in het project hadden gewild, maar de grote functionaliteiten zijn er door de samenwerking van ons allemaal.
